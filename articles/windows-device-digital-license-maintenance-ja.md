個人向け PC の Windows ライセンスは主に、メーカー製市販PCに組み込まれている **OEM ライセンス**、PC部品とのセット販売用の **DSP ライセンス** と、パッケージ版での販売の **リテール ライセンス** の3種類があります。
このうち **OEM ライセンス** と **DSP ライセンス** はメインボードやPC構成部品とWindowsライセンスのセット販売です。
ハードウェアと Windows ライセンスの組み合わせによる特別割引きのため、ライセンス認証後はメインボードやCPU等の主要部品の交換が出来ません。

しかし **リテールライセンス** は、特定の要件を満たすことで **少しずつ** ですが、部品交換が可能になります。
例えば自作PCや改造PCで **リテールライセンス** で認証している場合、ライセンス認証状態のまま、構成するハードウェアの一部入れ替えが出来ます。

Windows XP の頃はライセンスの継続に、どの様な部品交換が可能なのかの条件が公開されていましたが、今はその条件は非公開です。
そこで、これまでの個人的な経験を元に、現在の **リテールライセンス** を維持したまま、どの様に部品交換が可能なのかを確認してみました。

なお、自己流の僅かな回数の実験に基づいた結果なので、この方法があらゆる環境でうまく行くかどうかはわかりません。
そして重要なことは、2023年10月に Windows 7, Windows 8, Windows 8.1 から Windows 10, Windows 11 への無償更新が完全に終了しています。
従って Windows 8.1 以前のリテール版ライセンスでは、元のライセンスされた Windows のバージョンでだけ、部品交換が可能な点に注意してください。

## 目的

この記事の目的は自作PC等の部品交換時に、電話認証などの人手による再認証をなるべく避けて、取得済のデジタルライセンスを自動認証で維持するための考察です。
そして部品交換時の手順を明確にすることで、不注意な操作によるデジタルライセンスの消滅をなるべく避けることです。

典型的な事例として、CPU付マザーボード交換と、新規インストール時のディスク交換の2種類を取り上げます。
近年では BIOSの更新や設定変更だけで「大幅な部品変更」とみなされてデジタルライセンスを失う場合もあり、どの部品交換がライセンス維持に影響を与えるか不明な点も多いです。
従ってすべての状況を明らかにすることはできませんが、この２つの事例が何かの役に立てばと思っています。

以降は、PC自作や組み替え、Windows 新規インストールを5台ぐらいは経験済の方を対象にした、少し高度な内容です。
各種アカウントの設定、Microsoft アカウントとその保守方法、デジタルライセンスに関する詳細な説明は省いていますので、ご注意ください。

## 事前準備

部品交換をする場合は必ず、デジタルライセンスが正しく認証されて、（ライセンス認証の命綱の）Microsoftアカウントに紐付け（リンク）されていることを入念に確認する必要があります。
以降解説の手順では、認証前に部品の取り付けや取り外しをする場合があるため、マザーボード交換直前の部品構成状態で確認します。

Windows 10 では 設定→更新とセキュリティ→**ライセンス認証** で、**Microsoft アカウントにリンクされた ライセンス認証**となっている事を確認します。

*Windows 10 のリンク済ライセンス認証*

Windows 11 では、設定→システム→ライセンス認証 で、**ライセンス認証の状態** を開いて **Microsoft アカウントにリンクされた ライセンス認証**となっている事を確認します。

![画像](/images/windows-device-digital-license-maintenance-ja/71e99fc2-4074-4064-ad47-500abecf6271.png)
*Windows 11 のリンク済ライセンス認証*

*すべてのデバイスの表示をクリック*

**すべてのデバイスの表示**例です。拡張表示で詳細情報を確認できます。

ここで、**昔からよく知られている重要なこと**は、このデジタルライセンスの命綱とも言える Microsoftアカウントの登録済デバイスの表示内容が、なぜか時々変更されたり、消えてしまう等といった場合があることです。
またマシン情報が変更されたり、Windows 11 24H2 から 25H2 の様なリリースバージョン番号の更新時、登録バージョン番号反映されない場合などの不具合もあります。

情報が間違っている場合、Microsoftアカウントでの該当マシン名を選択して開き、右上の **このデバイスを削除** をクリックして削除します。

![画像](/images/windows-device-digital-license-maintenance-ja/QQ-DD.png)
*このデバイスを削除 をクリックして削除*

その後 PC側では、必ず次の手順で、Microsoft アカウントのデバイス再登録を行っておきます。
この手順は、PC構成の入れ替時に必須で、納得いくまで何回も再起動、ログインし直しを入念にして正しく登録してあることを確認します。

前述の様にブラウザで参照する **Microsoftアカウントの登録済デバイス表示** は管理用データーベースには問題がある様子で、結構な頻度で表示が不調になります。
その様な場合、1日程度待ってから再挑戦することをお勧めします。

## Microsoft アカウントのデバイス（再）登録手順

ブラウザでのMicrosoft アカウントのデバイス削除後は、該当の Windows PC で以下の手順で操作して、再登録します。

1. 目的のPCで操作して、登録対象の「Microsoft アカウント」にサインイン

2. そのまま「ローカルアカウント」でのサインインに切り替える

3. そのまま、Windows を再起動

4. 最後にサインインした「ローカルアカウント」で、該当PCにサインイン

5. そのまま「Microsoft アカウント」でのサインインに切り替える

6. そのままブラウザで、該当の「Microsoft アカウント」でサインインして、デバイスを確認

7. 治らない場合は、翌日以降に1番から再実行


## マザーボード交換時

マザーボード交換による再認証のコツは、交換対象のマザーボードとCPU以外の部品を極力流用することです。
とは言ってもオンボードにほとんどのデバイスが載っている現在では、最低でも C:ドライブのディスク（ストレージ）とDVDドライブ（光学ドライブ）を流用することで「マザーボードとCPUが交換されたが同じマシンだ」と思わせる様な努力が必要です。

可能であれば再認証時だけでも、ビデオカードも流用した方がいいです。Windowsがライセンス継続利用を許容する部品の変更範囲は明らかではありませんが、Windows XPの頃はリテール版でのライセンス流用可能なPC変更識別の条件としてCPU、Diskドライブ、MACアドレスとともに **DVD/CDドライブが同じ** という条件が公開されていました。
それで今でも迷信の様に、この光学ドライブ条件を積極的に「同じマシンの流用（更新）を主張」する目的で利用します。
マザーボード交換の際の物理 C:ディスクドライブの継続使用は有用です。場合によっては、再認証前の新規インストール（詳細手順は後述）も考えます。
デジタルライセンスを維持する上で重要なことは、以前使っていた環境と同じという物理ハードウェアやデバイスのヒントが一つもない、つまり**全く新しい環境ではどうやっても再認証されない**という事です。

[Microsoft サポート コミュニティの質問の回答](https://answers.microsoft.com/ja-jp/windows/forum/all/%E3%82%A2%E3%82%AF%E3%83%86%E3%82%A3%E3%83%99/42b09e7d-9081-466b-b49b-8d87b30775db)に書いた通り、この方法で3台のPCを組み換えましたが、いずれも再認証が出来ました。

補足として大事なことは、再認証時には、結構なトラブルがあるので、ライセンスサーバーが停止している旨のメッセージが出る場合には、ネットワーク周りの設定を良く確認する等して何回も再挑戦します。翌日になって認証される場合もよくあります。

なお組み換え後初めての電源投入ですんなりとライセンス認証されない場合は、ライセンス認証のトラブルシューティングから入って、表示される「このデバイス上のハードウェアを最近変更しました」をクリックして進みます。

## ディスク交換時

ディスクドライブ機種交換と Windows 新規インストールを同時に行った場合の再認証事例です。次の手順でした。

1. 交換対象ディスクドライブ以外は全て同一構成（同一マシン）とします。

2. マシン名、WORKGROUP、アカウント名、当然 Microsoftアカウント名を以前のマシンと同じにします。

3. 新規インストールの再認証用ディスクのOSレビジョン、リリースバージョン、Windows Update は再認証前と同じにします。

4. 未認証の新ディスク（新規インストール Windows）で立ち上げて再認証のためのトラブルシューティング実行時、最初は全く登録済デバイスが認識されず拒否され、ライセンスを購入せよとの警告が出ます。

5. 少し待ち、IPアドレス、プロキシ等のネットワークの設定の変更、Microsoft アカウントでのログインし直し、登録状況のWebで登録内容の確認、ローカルアカウントとの行き来等をしてみます。

ここでトラブルシューティングを何回かクリックしながら待っていると、やがて「ライセンス認証をする」趣旨のリンクが現れるので、それをクリックします。
そして「ライセンス認証が完了しました」の表示が出ます。
当然ながらマシン名を合わせているため、CPU マザーボード交換時の様に、このマシンですか？みたいなマシン名の確認はありません。
これで再認証完了です。
この時も確認しましたが、Windows 本体が入れ替わった際のデジタルライセンス再認証には、最低でも数十分程度の時間がかかるようです。

以上、あくまでも自己経験による事例紹介でした。繰り返しですが、動作保証はしません。リテール版ライセンス以外では利用出来ません。

## NVMe/M.2 付マザーボード交換時

最近のNVMe/M.2 付マザーボード交換では、昔の様に Cドライブだけ残して流用するといった様な、同一デバイスを使い回して、マザーボードだけの部品交換だと認識させること難しくなりました。
そうではどするかと言うと、光学ドライブと増設の Dドライブ（HDD）を流用して再認証に挑戦し、通らなかった場合は、後述の「60日間待つ」を利用する作戦をとります。

また、どの程度有効なのかは不明ですが、マザーボード入れ替えの際に同じ製造メーカーの似た様な型番、仕様の製品への交換だと、マザーボード交換時にライセンスが外れることが少ない気がします。
これはライセンスの再認証でコンピューター ハードウェア ID (CHID) を比較する場合、製造元IDやファミリID の差が出にくいかからかも知れないと、勝手に妄想しています。

参考：コンピューター ハードウェア ID (CHID) の使用
https://learn.microsoft.com/ja-jp/windows-hardware/drivers/dashboard/using-chids?WT.mc_id=WDIT-MVP-35878

## 60日間待つ

最近はこれらのライセンスを失わない工夫をしても、ライセンスの再認証が上手く動作しない場合を、何回か経験しています。
その様な場合の最も簡単で有効な方法は「60日間待つ」です。

これは大幅な部品変更をした場合のリテール版ライセンスの再認証時に使う方法ですが、最近はBIOS更新やBIOSの設定変更だけで「部品変更」と検知されてライセンスが無効になってしまう場合も増えているため、やむを得ない対処方法として、利用する機会が増えています。ライセンス認証がされていない60日間は、全く何も使えないわけではないため、以外と早く経ちます。覚えておくとよい手段です。
