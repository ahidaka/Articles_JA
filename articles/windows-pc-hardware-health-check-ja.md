Windows PC の不具合は、システムファイルのエラーやファイルシステムの損傷といったソフト的な問題の場合が多いのですが、メモリーやストレージ、ネットワークデバイスといったハードウェアの不具合、つまり故障が原因の場合もあります。
ここでは Windows のトラブル対処で重要な、ハードウェアの正常性検証の手順をまとました。

トラブル発生時、明らかに設定やソフトウェアが原因の問題だと確認出来ない場合は、以降に示すハードウェアの正常性検証を確認してから、問題への対応を行うべきです。Windows PC は機械なので、使用しているうちに部品に不具合や経年劣化が発生して、その結果ハードウェアの故障を引き起こすことがあります。

この様に、障害発生時にハードウェアの不具合を疑い、正常性を検証することは問題解決に重要です。
確認すべき項目は次の通りです。

- 初期不良
- 設置・運用・環境の人的ミス
- メモリ
- ハードドライブ （ディスク）
- 電源ユニット

また最後に [**トラブルシューティング**](#トラブルシューティング) と、ハードウェアが正常だった場合の **Windows PC のソフト的要因のトラブル解決方法** を含めた [**まとめ**](#まとめ) も示します。
これら[**トラブルシューティング**](#トラブルシューティング) と、**[ソフト的要因のトラブル解決**](#ソフト的要因のトラブル解決) を実行する方法は、前述のハードウェアの正常性検証よりも容易に実行出来るので、状況によってはこちらを先に試行する手順も考えられます。

## 初期不良

新品PCや電子機器を購入して1か月程度など、使い始めてまだ時間が経っていない場合に発生する **部品不良** が原因の不具合は、「初期不良」と呼びます。これはデジタル機器でたまに起きる障害として、販売店や専門家には広く知られています。

半導体等のデジタル機器で使用される電子部品は、製造後安定動作するまでに時間がかかるという特性があります。そのため民生用のデジタル機器は、購入後短期間の間に起きる初期不良には特別な対応をするという、PC 製造業界のルールがあります。
また「初期不良」は、その問題の製品の販売店が一時対応（トラブル対応における最初の窓口）をすることに対応方法が決まっています。
しかし対応内容は問題の状況、製品種類、販売店やメーカーによって異なります。一般には返金や代替品の提供といった、購入者が不利益にならない対応がされることになっています。ここで重要なことは、初期不良の不具合を解決出来るのは、購入者と販売店だけだという事です。

初期不良対応期間を過ぎた場合、あるいは初期不良対応について知らなかった場合、あるいはメーカー直販製品の場合は、各メーカーが用意している通常1年間程度の「無償保障」で対応することになります。

## 設置・運用・環境の人的ミス

前項のトラブルシューティングとも関係しますが、実際のハードウェアの不具合原因で圧倒的に多いのは、設置・運用・環境 に関する人的ミスです。

具体的にはコネクターの接続場所の間違い、各種ソケットの差し込み不良、電源の入れ忘れ、電池切れといった、軽微かも知れませんが重要な設置・運用上の人的ミスが、致命的な問題の原因である場合は多いです。
例えばワイヤレス・マウスの電池切れもこの手の人的ミスですが、全てのPCに「CMOS バックアップ バッテリー」と呼ぶ、5年程度の寿命のボタン電池があり、その電池切れが PC の不具合を引き起こす場合があることを知らない利用者は多いです。

環境の問題としては、AC電源ケーブルを奥まで正しく装着しないといった設置時の単純な問題から、AC電源供給電力の容量不足や、各種ネットワーク機器や Wi-Fi 等の無線ネットワークの設置ミスや電波の干渉といった前述のトラブルシューティングでは発見し難い問題の場合があります。

この様な実行環境と関わる人的ミスは、トラブル発生時に入念に検証することと、ある程度 Windows PCの設置・運用経験を積むことぐらいしかありません。昔、結果的には初期不良が原因で発生したトラブルで、大手PCメーカーに問い合わせたとき、「失礼ですがお客様、ACコンセントの電源を確認されたでしょうか？」と指摘されたことがあります。トラブル対応ではそういった基本的な確認の積み重ねが重要です。

## メモリ

一見正しく動作している様に見えるPCでも、実はメモリ異常であることは意外と多くあります。
最初のうちは正常動作していても、使っているうちにメモリ素子の経年劣化などが原因で動作不良や相性問題が発生する場合です。
SIMMと呼ぶメモリモジュールは簡単に入手・交換・増設できて便利ですが、正常動作する接続先を選んでしまう**相性問題**がある場合もあります。サーバー用マシン以外の多くのPCハードウェアは、動作中にメモリのエラー訂正や正常性検証をしないため、メモリ異常に起因する不具合が発生しても分かり難くなっています。

メモリの検査は、スタートメニューの **Windows ツール** に付属している **Windows メモリ診断** を使用して行うことが出来ますが、検証精度はそれほど高くありません。
高負荷時や特定パターンでのメモリアクセス エラーを見逃す場合があります。
ここでは、**Memtest86+** という PCメーカーでも使用しているソフトウェアツールを使って検査する方法を示します。以下に手順を説明します。

### 準備とMemtest86+のダウンロード

1. まず Windows PCと、初期化しても良い1MB以上の容量の USB メモリを用意します。
2. 次に [**https://memtest.org/**](https://memtest.org/) をブラウザで開き、画面右側の [**Download Latest Version の Windows USB Installer (32/64 bits)**]をクリックします。
3. **mt86plus_8.00_USB_Installer.exe** のダウンロードが始まるので、保存します。

![画像](/images/windows-pc-hardware-health-check-ja/86dl.png)
* Windows USB Installer (32/64 bits) を選択してダウンロード*

### Memtest86+のインストール

Memtest86+ は Windows などのオペレーティングシステム上で動作しません。
USB フラッシュメモリーに書き込んで、BIOS から直接起動して動かします。
ダウンロードした、mt86plus_8.00_USB_Installer.exe はこの、BIOS から起動する USB メモリを作成するためのツールです。インストール手順を示します。

1. 用意したUSBメモリーを装着後、入手した **mt86plus_8.00_USB_Installer.exe**（ファイル名はバージョンよって変わる）をクリックすると、ユーザーアカウント制御を確認後インストールが開始します。

2. **I Agree** をクリックしてライセンス確認します。  

![画像](/images/windows-pc-hardware-health-check-ja/m8-1.png)
*I Agree クリック*

3. 書き込み先のUSBドライブ文字を設定・選択して、**We will format ...** にチェック後 **Create** をクリックします。USBメモリーへの書き込みが始まります。 

![画像](/images/windows-pc-hardware-health-check-ja/m8-2.png)
*ドライブ文字入力とチェックして **Create***

4. Installation Complete 画面で **Next** をクリック後、次画面で **Finish** で完了です。  

![画像](/images/windows-pc-hardware-health-check-ja/m8-3.png)
*Next をクリック*

![画像](/images/windows-pc-hardware-health-check-ja/86fin.png)
*Finish で終了*

### Memtest86+の起動

書き込み完了した USBメモリを装着して、テスト対象のPCを起動します。
USBメモリを装着してPCの電源を入れ直すだけで起動する場合もありますが、環境によっては UEFI/BIOS の設定変更や、電源ON時に「F1」等の特殊キー押下により一時的にブートデバイスを USB Flash に変更する操作が必要です。

重要なことは、このとき **UEFI BIOS で一時的にセキュアブートを無効にして起動し直しておく必要** があります。

セキュアブートを無効にする設定は、機種によって異なるので、注意が必要です。
テストを完了後は、元の設定に戻しておきます。

![画像](/images/windows-pc-hardware-health-check-ja/PXL_secureboot.jpg)
*BIOSのセキュアブート無効化設定画面例*

### テスト開始

メモリテストは、PC 起動時に自動的に起動して進みます。
メモリー容量や CPU などの環境にもよりますが、一回目のテストが終了するまで **30分から2時間程度** かかります。途中で終了する場合は、**ESCキー** を押します。

![画像](/images/windows-pc-hardware-health-check-ja/PXL_20260114_00start.jpg)
*開始画面の例*

次に示す様に **Pass: のカウンターが 1回以上になり、**PASS** の大きな文字が表示されたたらば、そのPCのメモリは問題無いです。**PASS** の大きな文字は表示後直ぐ消えるのでご注意ください。

![画像](/images/windows-pc-hardware-health-check-ja/PXL_20260114_01pass.jpg)
*PASS画面の例*

メモリ環境に異常がある場合は、次の様に画面上に赤い背景でエラーが報告され、**FAIL** の大きな文字が表示されます。**PASS** の大きな文字は表示後直ぐ消えるのでご注意ください。**FAIL** が表示されるメモリは故障なので、試した範囲では他のPCでも使えませんでした。

![画像](/images/windows-pc-hardware-health-check-ja/PXL_20260114_02fail.jpg)
*FAIL画面の例*

メモリのテストツールは、前述の **Windows メモリ診断** を始め  **Memtest86+** 以外にも同種のMemtest系ツールなどをはじめ、いくつかあります。しかしこの **Memtest86+** は最新のテスト内容と環境向けに結構な頻度で更新されていて、同じ環境で試したときには他のツールよりもより早く、より厳しく問題を発見することを確認しています。知っておくとよいツールです。

## ハードドライブ

現在のハードドライブ（SSD/HDD/NVMe などのディスクドライブ）は、それ自体が小さなコンピューターです。
ドライブのRead/Writeには記録メディア（磁性体やフラッシュメモリ）のエラーがつきものです。
ドライブでは記録データを保全しながらメディアの寿命を伸ばすためエラー訂正、リトライ（再読み出し）のほか、バッドブロックと呼ぶ不良な記憶領域を記録して避ける仕組みが動作しています。またフラッシュメモリーでは、個々のメモリー素子の寿命管理も重要な機能です。

各ドライブはコンピューターですから、特殊な管理領域に過去の動作やエラーの記録を持ち、保全しています。SMART (S.M.A.R.T. Self-Monitoring, Analysis and Reporting Technology)と呼ぶ、この管理情報を参照する方法で、現在のドライブの状態や過去のエラー履歴を確認することができます。

Windows 上で SMART を確認するツールはいくつかあります。ここでは使い易く、結構な頻度で最新の状況に合わせた更新が行われている **CrystalDiskInfo** について説明します。

### ダウンロード

様々なサイトで入手可能ですが、ここでは Microsoft ストアで入手する方法を示します。

Microsoft ストアで「CrystalDiskInfo」の名前で検索すると、次の様な結果が表示されます。

![画像](/images/windows-pc-hardware-health-check-ja/crystal-1.png)
*検索結果表示*

目的の診断ツールは最初に表示される、**CrystalDiskInfo** です。これと同じく3番目に表示されるキャラクターイラスト付きの「CrystalDiskInfo Aoi Edition」と「CrystalDiskInfo Shizuku Edition」も、機能が同じなので同様に利用することが可能です。
2番目に表示される CrystalDiskMark は、同じ作成者によるストレージのベンチマーク評価ツールです。これらは、[ツールの開発者である ひよひよ氏](https://crystalmark.info/ja/information/profile/) が運営する [本家のダウンロードページ](https://crystalmark.info/ja/download/) などからも、入手可能です。

### 起動と検査

以下に起動後の画面例を示します。表示するだけのツールなので、検査対象ドライブを選択するだけです。

現在表示しているドライブに問題が無い場合は、この様に水色の背景で **正常** と表示されます。
**正常** の後の 97% は NAND型フラッシュメモリーが使用している残りの素子の比率、すなわち残りの寿命です。

![画像](/images/windows-pc-hardware-health-check-ja/crystal-2.png)
*NVMeドライブの実行例*

HDD の場合、バッドブロック（不良セクタ）の発生状況と、回復不能セクタ数といった状況が表示されます。
近年のHDDは製造単価が下がり、性能も向上しているため、代替処理が必要な状況の場合は、黄色の **注意** 文字が表示されます。

以下は、代替セクター（バッドブロックの置き換え）が発生しているHDDの状態画面です。健康状態が**注意**となり、黄色のアイコンで表示されます。もし代替処理保留中のセクタ数がゼロではない場合には、今後**データの損失の可能性**があるため注意が必要です。ドライブが不良な場合やメディアが高温の場合には、**赤色アイコン**で異常を伝えます。各ドライブの表示項目と意味は、そのドライブが持つ SMART 機能に依存しています。

![画像](/images/windows-pc-hardware-health-check-ja/cry-x.png)
*注意画面の例*

代替処理保留中のセクタがある場合、該当ドライブの交換が最良の対策の１つです。他の方策としては、問題があるドライブのデータをバックアップした上で、物理フォーマットやスキャンツールを使用したドライブ・スキャン等をして、保留中のセクターをゼロにする方法があります。

また最近増えて来た SSD/NVMe/M.2 ドライブでは、デバイスや内部素子の不具合が発生しているのにも関わらず、水色表示、表示の数値がめちゃくちゃ、結果が一切表示されないといった場合があります。
その様な場合は、パーティションの作成し直し、物理フォーマットのし直し、ベンチマーク評価といった他の方法を使用して、正常動作を確認するしかありません。ご注意ください。

## 電源ユニット

ケーブル類や環境も正しい、メモリー、ディスクも異常が無いといった場合に、PC 本体接続の電源ユニットが問題がある場合があります。
PCの電源ユニットはノート型でもデスクトップ型でも経験上、PC部品の中で一番経年劣化や不良、容量不足等のトラブルが多い部品です。

電源ユニットの異常は一般的に「電源が入らない」「運用中に突然電源が切れる」「異音がする」「異臭がする」といった症状があるため、分かり易いです。また電源ユニットは経年劣化が起きる場合があるコンデンサーが多数使われていることから、5年や10年の使用期間で考えた場合、交換が必要な一種の消耗品であるとも言えます。
自作パソコンで古い電源ユニットを使い回している方は、注意が必要です。

厄介なのは、正常動作している様に思えても、電源ユニットの不良や容量不足が原因の問題が発生している場合があることです。デスクトップPCでは、グラフィックボードを追加・交換する場合がありますが、その際は電源容量を確認する必要があります。

デスクトップ型自作パソコンであれば、別の電源ユニットに交換して試すのことが、手っ取り早い電源ユニット不良の問題解決方法です。消費電力を測るワットチェッカーを使用して、消費電力が予定の範囲内か確認する方法や、負荷運用中にデジタルテスターを使用して、内部供給の直流 5Vと12Vが正しく出力されているか確認する方法などがあります。

## まとめ

ここまで説明して来た様に Windows で発生した問題は、初期不良かどうかを含めて、ハードウェアの正常動作を確認した上で問題の原因を見極めて対策するのが定石です。
しかし Windows には、運用上のトラブルを解決するための「トラブルシューティング」用のツールが組み込まれているので、これを利用して発生している問題を診断する方法があるので、これを試す方法があります。

### トラブルシューティング

トラブルシューティングは、設定パネルのシステム > トラブルシューティング から **その他のトラブルシューティング ツール** を選択して実行します。

![画像](/images/windows-pc-hardware-health-check-ja/TroubleS.png)
*トラブルシューティング*

オーディオ、プリンター、Bluetooth、カメラ といったハードウェア関連の診断機能がありますが、基本的には設定やソフトウェア関連の診断機能が中心です。

![画像](/images/windows-pc-hardware-health-check-ja/TroubleSS.png)
*その他のトラブルシューティング*

より詳しい情報は、Microsoft Learn の [**Windows クライアントのトラブルシューティング**](https://learn.microsoft.com/ja-jp/troubleshoot/windows-client/welcome-windows-client?WT.mc_id=WDIT-MVP-35878) のページでも解説しています。

それでも残念ながら「メモリー」「ストレージ」といった重要な部品については、ここまで説明して来た様な厳密なハードウェアの正常性診断は実行しません。
あくまでも Windows の標準機能で判断出来る、限られた範囲でのトラブルシューティングの実行と考えておいた方がよいです。

一方でトラブルシューティングの各ツールは、ハードウェアやソフトウェアに限らず、見落とされがちな Windows のプライバシーとセキュリティや、プログラムの互換性調査といった機能が提供されるので有用です。
さらに新しい法式の通信プロトコルへの対応など、少しずつですが Windows の進化とともに更新されているので、トラブル対処方法の一つとしては知っておくべきです。

### ソフト的要因のトラブル解決

ここからはソフト的要因のトラブル調査・解決手順を示します。
特にストレージ異常が疑われる場合は、前述の **CrystalDiskInfo** で **SMART** の異常がないことを確認後、ディスクスキャン で論理エラーを検査・修復、システムファイルチェッカーによる検証、が定番の修復作業です。
以降にこの一連の手順を示します。

#### クリーンブート

[**クリーンブートで起動**](https://support.microsoft.com/ja-jp/topic/windows-%E3%81%A7%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3-%E3%83%96%E3%83%BC%E3%83%88%E3%82%92%E5%AE%9F%E8%A1%8C%E3%81%99%E3%82%8B%E6%96%B9%E6%B3%95-da2f9573-6eec-00ad-2f8a-a97a1807f3dd) に解説と詳細な手順が記述されているので、それに従います。

最初はすべてのスタートアップ項目を無効にする設定から始めて、少しずつ動作を確認して行きます。
前述の「ソフト的要因のトラブル解決」を目的とした、一連実行手順の場合は、すべて無効で機動したまま、ディスクスキャンに進みます。

#### ディスクスキャン

前項、[**クリーンブート**](https://support.microsoft.com/ja-jp/topic/windows-%E3%81%A7%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3-%E3%83%96%E3%83%BC%E3%83%88%E3%82%92%E5%AE%9F%E8%A1%8C%E3%81%99%E3%82%8B%E6%96%B9%E6%B3%95-da2f9573-6eec-00ad-2f8a-a97a1807f3dd) で表示されているすべての、スタートアップサービスを無効にした後、Cドライブのプロパティ、ツール タブ でドライバーのチェック、ドライブのスキャン を実行します。
ここでエラーが出る場合は、出なくなるまで繰り返します。
クリーンブート環境で実行するのは、セキュリティ関連を含む常駐スタートアップサービスの影響排除が目的です。

#### システムファイルチェッカー

[**クリーンブート**](https://support.microsoft.com/ja-jp/topic/windows-%E3%81%A7%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3-%E3%83%96%E3%83%BC%E3%83%88%E3%82%92%E5%AE%9F%E8%A1%8C%E3%81%99%E3%82%8B%E6%96%B9%E6%B3%95-da2f9573-6eec-00ad-2f8a-a97a1807f3dd) で起動後、引き続いて管理者権限でコマンドプロンプトを起動して、[**システムファイルチェッカー の実行**](https://support.microsoft.com/ja-jp/windows/windows-%E3%81%A7%E3%81%AE%E3%82%B7%E3%82%B9%E3%83%86%E3%83%A0-%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB-%E3%83%81%E3%82%A7%E3%83%83%E3%82%AB%E3%83%BC%E3%81%AE%E4%BD%BF%E7%94%A8-365e0031-36b1-6031-f804-8fd86e0ef4ca) 手順に従って、チェックを行います。

システムファイルチェッカーの実行は、最新のシステムイメージをネットワークから取得する **DISM.exe** 実行部と、実行対象PCのシステムファイルの検証を行う、**sfc /scannow** 実行の二つの手順に分かれています。

もし **sfc /scannow** 実行後、コマンドプロンプトでエラーや警告が表示される場合は、それらが無くなるまで、繰り返し **sfc /scannow** を実行します。

### 新しいユーザーアカウント

新しいローカルアカウントのユーザーアカウントを作成して、そのアカウントでサインインして動作を確認します。これで問題が無い場合は、ユーザープロファイルの損傷が原因です。

### セーフモード

セーフモード起動して動作を確認します。以下のドスパラの記事がわかり易いので紹介します。
これで問題が無い場合は、スタートアップサービスで組み込み済のネットワークやドライバーや、サービスが原因です。 

https://www.dospara.co.jp/cts_str_pc_safemode.html

### 修復インストール

さらに強力な解決策としては、**Windowsの上書きインストールや修復インストール** があります。正常なハードウェアであれば、再インストールして動作しない Windowsはありません。
ストレージが高速化している現在、Windows インストールはそれほど時間がかかるものでは無くシンプルな方法なので早目に試しておくべきです。

### 最後に

Windows PC は一種の機械なので、故障したり、トラブルが発生したりする可能性があります。
しかし機械なので、部品交換を含めて適切な処置や修理を行なうことで、必ず正常動作する様になります。
不具合が発生しても諦めずに頑張ってください。
